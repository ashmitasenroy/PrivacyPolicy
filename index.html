RevuAI — Privacy and Data Protection Policy

A clear, short explanation of what user code is sent to Google Gemini (the AI provider) and why RevuAI uses it.

Intro

RevuAI is an independent browser extension that uses Google Gemini to analyze code submitted by users. This page explains what data we send, why we send it, and how we handle it.

What data we send

We want users to be clear and comfortable about their data.

Data sent to the AI service:
  • The user's source code or code snippets submitted for automated review.
  • Optional metadata: programming language and the user's explicit prompts or instructions.

We do not intentionally send any personal identifying information or secrets (like API keys or passwords) from your code. Always review your code before submitting.

RevuAI does not collect, use, or transmit personal information, account data, or browsing activity.

Why we send it

Your code is sent to Google Gemini for the following purposes:

  • To perform automated code review, identifying potential issues and areas for improvement.
  • To generate suggestions for better code structure, efficiency, and clarity.
  • To provide clear explanations of detected issues and proposed solutions.

This process helps you quickly improve your code quality and learn best practices.

Do we store your code?

We are explicit and plain about data retention:

We do not store your submitted code on our servers.

Your code is transmitted to the third-party AI provider (Google Gemini) for processing and is not persisted by RevuAI beyond the duration necessary to obtain a response. We do not retain logs of user-submitted code.

Third-party policy & links

We rely on Google Gemini (a Google AI product) for code analysis. You should review how Google handles data:

  • Google Privacy Policy — general privacy information.
  • Google Cloud Generative AI Data Governance — details on how Google handles data used with generative AI models.
